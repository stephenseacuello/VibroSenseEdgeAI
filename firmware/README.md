# `firmware/` — Arduino Nano 33 BLE Sense

The ISA-95 Level 1 tier. Reads the LSM9DS1 IMU at 952 Hz, builds 256-sample windows, runs TFLite Micro inference, and publishes state + raw windows over BLE.

See [PROJECT_PLAN.md §11](../PROJECT_PLAN.md#11-firmware-plan-arduino-nano-33-ble-sense).

## Layout

```
firmware/nano33/
├── nano33.ino                 # main sketch (sample, infer, publish, watchdog)
├── src/
│   ├── inference.{h,cpp}      # TFLite Micro interpreter (loads model.h, runs INT8 inference)
│   └── ble_service.{h,cpp}    # GATT service, raw_window encoder, watchdog, LED
└── model/model.h              # generated by `make export` after training
```

## Prerequisites

```bash
brew install arduino-cli       # macOS
arduino-cli core update-index
arduino-cli core install arduino:mbed_nano
arduino-cli lib install ArduinoBLE
arduino-cli lib install "Arduino_LSM9DS1"
# TFLite Micro for Arduino — either of:
arduino-cli lib install --git-url https://github.com/tensorflow/tflite-micro-arduino-examples
# or
arduino-cli lib install "Chirale_TensorFlowLite"
```

## Build and flash

```bash
make firmware                  # compile only
PORT=/dev/cu.usbmodem14101 make flash
```

`arduino-cli board list` will tell you the right port. Common values: macOS = `/dev/cu.usbmodemXXXX`, Linux = `/dev/ttyACMX`.

Debug builds with per-window timing on serial:

```bash
arduino-cli compile --fqbn arduino:mbed_nano:nano33ble \
    --build-property "build.extra_flags=-DDEBUG_TIMING=1" firmware/nano33
```

Then watch with `arduino-cli monitor -p $PORT`.

## BLE GATT layout (ADR-0001, ADR-0004)

Advertised as `VibroSense-Nano`. Service + four characteristics:

| Characteristic | UUID                                      | Props        | Format |
|---|---|---|---|
| `state`        | `…def1`                                   | read, notify | UTF-8 JSON, `schema_ver=1` (≤ 96 bytes) |
| `mode`         | `…def2`                                   | read, write  | uint8: `0=INFER`, `1=CAPTURE` |
| `version`      | `…def3`                                   | read         | UTF-8 build identifier |
| `raw_window`   | `…def4`                                   | notify       | chunked binary frames per ADR-0004 |

UUIDs are placeholders (`12345678-…`). Regenerate via `uuidgen` and freeze in [ADR-0001](../docs/decisions/0001-ble-payload-schema.md) before tagging `v0.2`.

## Status LED (RGB on the Nano 33 BLE Sense)

| State        | LED                |
|--------------|--------------------|
| advertising  | blue solid         |
| connected    | green solid        |
| CAPTURE mode | yellow (R+G) solid |
| error        | red blinking       |

Implemented in `StatusLed::{begin, set, update}`. The blink for `ERROR` is non-blocking — `StatusLed::update()` is called every loop tick.

## Watchdog

If a central is connected but no notify has gone out in 30 s, the main loop calls `BLEService::softReset()` which tears down advertising, restarts the radio, and re-adds the service. The LED returns to `ADVERTISING`. This recovers from the BLE stack getting stuck without a full hard reset.

## Memory and latency

See [PROJECT_PLAN.md §4](../PROJECT_PLAN.md#4-performance-budgets). Targets:

- Flash usage ≤ 70% of 1 MB
- SRAM usage ≤ 70% of 256 KB
- INT8 model ≤ 30 KB
- TFLite Micro tensor arena 24–40 KB (tune at Wk 5)
- p95 inference < 250 ms per window

Verify with the `arduino-cli` "Sketch uses" output (ATP-02) and `DEBUG_TIMING` over serial (ATP-01).

## Bringing up a real model

Out of the box this repo ships a placeholder [`model/model.h`](nano33/model/model.h) (`len = 0`) so the firmware compiles + flashes immediately. To enable real inference:

1. Capture labeled data with the Nano in CAPTURE mode:
   ```bash
   make capture ARGS="--class-label HEALTHY --operator amuszynski --speed 2 --duration-s 300"
   # repeat per class until ≥ 30 min per class
   ```
2. Train, quantize, and regenerate `model.h`:
   ```bash
   make train    ARGS="ml/data/raw/session.parquet --epochs 60"
   make quantize ARGS="ml/artifacts/cnn.keras --rep-data ml/data/raw/session.parquet"
   make export   ARGS="ml/artifacts/cnn.tflite"
   ```
3. Recompile + flash:
   ```bash
   make firmware && PORT=/dev/cu.usbmodem14101 make flash
   ```
4. Verify against the ATP suite: ATP-01 (latency, via `-DDEBUG_TIMING=1`), ATP-02 (memory, via `arduino-cli compile` size output), ATP-05 (`make eval`).
5. Record results in the `v0.5` tag notes.

If `Inference::begin()` returns false after a real model is exported, the most common cause is that `AllocateTensors()` ran out of arena. Bump `kTensorArenaSize` in [`src/inference.cpp`](nano33/src/inference.cpp) (default 32 KB) until allocation succeeds; this is the build-report number that goes into the memory budget table.

## Troubleshooting

- **"IMU init failed"** — check that the `Arduino_LSM9DS1` library is installed and the board is genuinely a Nano 33 BLE Sense (not the plain Nano 33 BLE; the IMU is on the Sense variant only).
- **No BLE advertising** — Soft Device may be stuck after a flash; press reset twice quickly to enter bootloader, re-flash, then power-cycle.
- **Drift on the gravity axis** — confirm the IMU is rigidly mounted and the sensor's z-axis is vertical. The capture tool's quality check flags mean |a_z| outside [0.7, 1.3] g.
